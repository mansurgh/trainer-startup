# Trainer#1 — Visual and Functional DNA

## 1. Core Identity

**Trainer** is a next-generation AI-powered fitness and nutrition application that combines modern technology, luxury design, and motivational intelligence.  
The visual identity of Trainer represents **discipline, elegance, and futuristic simplicity**.

### Visual Direction
- Minimalistic but powerful: clean, symmetrical compositions with bold typography.
- Mood: black and chrome aesthetic, with glass reflections and subtle neon glows.
- Energy: confident, determined, motivating — “better than a real one.”
- Inspiration sources: Apple Design, Mercedes G-Class interior, premium gym architecture.

---

## 2. UI Layout System

### Main Screens
1. **Onboarding / Welcome**
   - Full-screen gradient or dynamic background.
   - Text: “Trainer” + tagline “Better than a real one.”
   - Smooth fade transitions.
   - Button: “Get Started.”

2. **Sign In / Sign Up**
   - Minimal black background, glowing text fields.
   - Rounded input boxes with subtle border light.
   - “Continue with Apple / Google” buttons below.

3. **Body Analysis (Photo Upload)**
   - Two-step layout: text instructions + centered card.
   - Option to upload or skip.
   - Soft gradient shadow under the button.

4. **Dashboard (Main)**
   - Personalized daily workout summary.
   - Visual: 3D muscle silhouette with highlighted active muscles.
   - Floating button to open AI chat.
   - Tabs below: *Workouts*, *Nutrition*, *Profile*, *Chat*, *Settings*.

5. **Workout Screen**
   - Timer widget (square, glassmorphic).
   - Video or camera input area (for form correction).
   - Exercise list with sets and reps.
   - Button: “Start Recording Technique.”

6. **AI Chat (Trainer Assistant)**
   - Dark chat bubbles, Apple-style layout.
   - Left side: AI (silver glow). Right side: user (blue glow).
   - Messages appear with fade-in animation.
   - Context memory: workout history, nutrition, progress.

7. **Nutrition Screen**
   - Daily meal plan layout.
   - Top: Calorie summary and macros.
   - Cards for breakfast, lunch, dinner.
   - Option to upload fridge photo for AI meal generation.

8. **Profile**
   - Avatar / Body silhouette (dynamic progress).
   - Stats: weight, BMI, streaks, “victory %.”
   - Graph: workouts per week.
   - GitHub-style activity grid for tracking progress.

9. **Settings**
   - Simple list layout with rounded tiles.
   - Toggles: Notifications, Dark mode, Sync, Privacy.

10. **Subscription / Paywall**
   - Full dark background, spotlight on 3D Brabus G-Class model.
   - Text: “Sorry, but I want it.”
   - Benefits list + plan cards below (monthly / annual).
   - Buy button with strong call-to-action glow.

## 3. Color Palette & Typography

### Colors
| Name | HEX | Description |
|------|------|-------------|
| Primary | #000000 | Core black background |
| Secondary | #111111 | Slightly lifted black for depth |
| Accent | #00FFC2 | Futuristic neon turquoise |
| Glow | #E91E63 | Active interaction glow |
| Text | #FFFFFF | Main text |
| Subtext | #B3B3B3 | Secondary info |
| Warning | #FF0033 | Errors / intensity |

### Typography
- **Primary Font:** SF Pro Display or Inter
- **Heading:** SemiBold 28–34px
- **Body Text:** Regular 16–18px
- **Buttons:** Medium 18px, uppercase
- **Numbers / Stats:** JetBrains Mono (for precision aesthetic)

---

## 4. Visual Effects & Motion

- **Glow:** Subtle turquoise or magenta edge lighting.
- **Shadow:** 30% opacity blur, 20px spread.
- **Animation:** Spring transitions, 0.4s ease-in-out.
- **Micro-motions:** Button pulses, progress loaders rotate with inertia.
- **Depth:** Use glassmorphism for panels (transparency 20%, blur 8px).

## 7. Developer Notes (for MVP)

- Framework: Flutter (Dart) with Firebase or Supabase backend.
- Components: use reusable `TrainerButton`, `TrainerCard`, `TrainerTile` widgets.
- Font sizes: responsive using `MediaQuery.textScaleFactor`.
- API calls: use Claude 4 Sonnet or GPT-5 for AI responses.
- Code style: consistent with clean architecture (MVVM).
- State management: Riverpod or Provider.
- Dark mode as default.
